# Acceptance Criteria Template

Acceptance criteria format used across all story and bug templates.

```
## Acceptance Criteria
- [ ] <AC 1>
- [ ] <AC 2>
...

## Notes
<Edge cases, open UX questions, dependencies, constraints>
```

**Acceptance Criteria:** checkboxes. Each AC must be observable, testable, verifiable by non-engineer:
- `- [ ] When <condition>, the user sees/can/cannot <observable outcome>`
- `- [ ] The system <measurable behavior> when <condition>`
- `- [ ] <Feature> is only accessible to <user type>`

**Notes:** edge cases, open UX questions, dependencies on other stories, constraints. Use `link_to(id)` for inter-story references.
